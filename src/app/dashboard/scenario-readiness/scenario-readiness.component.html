Overview

The primary purpose of readiness grading is to determine if a given Scenario is ready to ship and meeting the goals we set out to achieve.

The following criteria is used to determine which Scenarios to display and grade;

  • Scenario State is not equal to Cut
  • Scenario is tagged for Readiness within a specific release, e.g., EDGE_RS2_READINESS, EDGE_RS3_READINESS, etcetera
  • Scenarios tagged with EDGE_CORE_SCENARIO_READINESS are also included as legacy scenarios
  • Search filters (Area Paths, Release, etcetera)

Grading Details

For all graded columns, the following formula is used to calculate the overall score.

(( [# of green items] * 1.0 ) + ( [# of yellow items] * 0.5 )) / [# of items excluding gray items]

State

Tracks which scenario are committed, completed vs proposed. Scenario is committed once scenario and scenario assessment spec reviews are completed and deliverables are identified.

Possible States

Green, Red, Gray

Grading Details

The Scenario State field is used to determine this grade.

Red

The grade is Red if;

  • State = Proposed

Green

Otherwise, the Grade is Green if;

  • State = Committed

Scenario Validation

owners: David Rubino & Joon Chang

All (applicable) user facing scenarios will be manually tested and assessed by vendors to identify issues impacting customer experiences.

Possible States

Green, Yellow, Red, Gray (Not Applicable), Gray (Not Yet Graded)

Grading Details

Both the Scenario and related Bugs are used to determine this grade.

Gray (Not Applicable)

The grade is Gray (Not Applicable) if;

  • Scenario Tags contain SCENARIO_VALIDATION_NA

Gray (Not Yet Graded)

The grade is Gray (Not Yet Graded) if;

  • Scenario Tags contain SCENARIO_VALIDATION_TBD

Red

Otherwise, the grade is Red if any related bugs meet the following criteria;

  • Contains the tag SCENARIO_VALIDATION_RED

Yellow

Otherwise, the grade is Yellow if any related Bugs meet the following criteria;

  • Contains the tag SCENARIO_VALIDATION_YELLOW

Green

Otherwise, the grade is Green.

Completeness

Once entering a stabilization phase, we’d expect all feature deliverables should be completed.

Possible States

Green, Yellow, Red, Gray (Not Applicable)

Grading Details

For a given Scenario, find related Deliverables that meet the following criteria;

  • State <> Cut
  • Product Family = Windows
  • Related Deliverables will also be constrained to a specific release if a release is specified as search criteria

Green

The grade is Green when all related Deliverables meet the following criteria;

  • State = Completed

Yellow

Otherwise, the grade is Yellow when one or more Deliverables meet the following criteria;

  • State <> Completed
  • Rank > 100 or Rank contains no value

Red

Otherwise, the grade is Red when one or more Deliverables meet the following criteria;

  • State <> Completed
  • Rank <= 100

Gray (Not Applicable)

Otherwise, the grade is Gray (Not Applicable)

High Impact Bugs

Tracks all engineering signoff blocking bugs impacting product health and user experiences.

Possible States

Red, Green

Grading Details

Red

The grade is Red if any related bugs meet any of the following conditions;

  • Blocking = Engineering Sign-off

    * note that high impact bugs are expected to encompass;

    • Hot Bugs (i.e., Hot Bug = Approved)
    • Edge Crit bugs, where Bug Tags contain both EDGE_CRIT and EDGEAPPROVED
    • Bugs that contain the tag EDGEREL_RED

Green

Otherwise the grade is Green.

Telemetry (RQV only)

owners: Will Frasier & Karthick Kumar Judun Gopichandranath

Tracks the product perf, reliability, accessibility and usage health leveraging telemetry data.

Possible States

Green, Yellow, Red, Gray (CTQ Not Required)

Grading Details

For a given scenario, we determine the number of related CTQ Measures and determine their states by looking at incoming telemetry data from Mission Control. This yields the number of measures that are passing, failing, or need more data.

Gray (CTQ Not Required)

The Scenario meets one of the following conditions;

  • Contains the tag CTQ_TESTCVG_EXIST
  • Contains the tag CTQ_TESTCVG_NEEDED

Red

The grade is Red if;

  • One or more CTQ Measures are failing

Yellow

Otherwise, the grade is Yellow if;

  • One or more CTQ Measures have insufficient data or no data

Green

Otherwise, the grade is Green.

Customer Feedback

owners: David Rubino

For a given scenario, we look at active feedback items associated with a scenario and make a determination if any of the feedback items should cause the scenario to be yellow or red.

Possible States

Green, Yellow, Red

Grading Details

To determine this grade, we look at all related Bugs where;

  • Bug State = Active
  • Issue Type = Data

Related Bugs are triaged into the following categories that are then used in grading;

  • Red = Bug Tags contain FEEDBACK_RED
  • Yellow = Bug Tags contain FEEDBACK_YELLOW

Red

The grade is Red if;

  • One or more related Bugs fall into the Red category

Yellow

Otherwise, the grade is Yellow if;

  • One or more related Bugs fall into the Yellow category

Green

Otherwise, the grade is Green.

Accessibility

owners: Bill Yang & Rory Fairweather

Tracks the health of accessibility.

Possible States

Green, Yellow, Red, Gray (Not Applicable)

Grading Details

Related Bugs are triaged into the following categories that are then used in grading;

  • Red = Bug Tags contain ACCSELFRED
  • Yellow = Bug Tags contain ACCSELFYELLOW
  • Lime = Bug Tags contain ACCSELFLIME

Gray (Not Applicable)

The grade is Gray (Not Applicable) if;

  • Scenario Tags contain EDGE_READINESS_ACCESSIBILITY-NA

Red

Otherwise, the grade is Red if;

  • One or more related Bugs fall into the Red or Yellow categories

Yellow

Otherwise, the grade is Yellow if;

  • More than five related Bugs fall into the Lime category

Green

Otherwise, the grade is Green.

Craftsmanship

owners: Danielle Ellbogen & Michael Thornhill

Tracks the attention to detail signals a premium product that you are proud to own.

Possible States

Green, Yellow, Red, Gray (Not Applicable), Gray (Not Graded)

Grading Details

The Craftsmanship grade is determined by looking at both the Scenario and related Bugs. A Bug tag prefixed with ‘c:’ represents its current Craftsmanship Score, e.g., “c:45” has a score of 45 which is then used to triage bugs into the following categories;

  • Score < 50
  • Score < 80
  • Score >= 80

Bugs are ignored and not triaged if their Resolution field contains one of the following values;

  • Won’t Fix
  • Duplicate
  • External

Gray (Not Applicable)

The grade is Gray (Not Applicable) if;

  • Scenario contains the tag EDGE_READINESS_CRAFTSMANSHIP_NA

Red

Otherwise, the grade is Red when any of the following conditions are met;

  • One or more Bugs have a score < 50
  • More than three Bugs have a score < 80

Yellow

Otherwise, the grade is Yellow when any of the following conditions are met;

  • One or more Bugs have a score < 80
  • Nine or more Bugs have a score >= 80

Green

Otherwise, the grade is Green if any of the following conditions are true;

  • There are less than 9 Bugs scored >= 80
  • There are zero Bugs (scored or not)

Gray (Not Graded)

Otherwise, if there are bugs (scored or not) and the Scenario tag CRAFTSMANSHIPREVIEWED exists, the grade is Gray (Not Graded).

Risk Assessment

owners: PM scenario owners

Tracks the risk assessment of scenario.  PM scenario owners to review all the scenario readiness metrics to determine if their scenarios can confidently land for RS3 with high quality.

Possible States

Green, Red, Gray (Not Applicable)

Grading Details

The value of the Scenario’s Risk Assessment property is used to determine this grade.

Red

The grade is Red if;

  • Risk Assessment = Off Track

Yellow

Otherwise, the grade is Yellow if;

  • Risk Assessment = At Risk

Green

Otherwise, the grade is Green if;

  • Risk Assessment = On Track

Gray (Not Graded)

Otherwise, the grade is Gray (Not Graded).
